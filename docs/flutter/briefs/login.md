## Current Flutter (divergence)

flutter_app/lib/app/screens/login_screen.dart already mirrors the web COMPOSITION closely: Scaffold with AppColors.bg, centered SingleChildScrollView → SizedBox(width:380) → sc.Card(filled, fillColor AppColors.surface, borderColor AppColors.line, radius AppSpacing.radiusLg=16, padding H36/V48) — L81-97. Inside a Column(stretch): "Watchparty" wordmark (13px w700 tracking -0.1, centered) — L102-111; 44px gap; "Welcome back" (22px w700 tracking -0.3 height1.15) — L113-122; "Sign in with your Jellyfin account" (15px w500 AppColors.dim) — L124-131; 28px gap; two AppTextFields (Username autofocus, Password obscure) — L133-147; a conditional error box with Reveal animation, colors 0x1FE0655E fill / 0x59E0655E border / AppColors.red text, 13.5px w500, radius 8 — L148-175 (matches web error box exactly); primary AppButton with busy spinner and "Signing in…"/"Sign in" — L177-183. Auth flow is fully wired and matches web semantics: flutter_app/lib/state/auth_provider.dart AuthNotifier.login() → ApiClient.login (cookie jar), restore() probes GET /api/auth/me at boot with an `initialized` flag the router waits on, logout() calls ApiClient.logout — auth_provider.dart:45-76. This functional layer is correct and must be preserved. DIVERGENCES: (1) THEME — AppColors (tokens.dart:8-30) is a HARDCODED dark-only monochrome ramp (bg 0xFF0A0A0B). Web login is theme-driven via CSS vars and the design guide (docs/watchparty-design/README.md:50-71) MANDATES three persisted modes (Light ~#f7f7f7, Balanced, Dark), all readable. Flutter login can only render dark — the forbidden "dark-only monochrome" old design. (2) FIELD LABELS — web labels are UPPERCASE JetBrains Mono 11.5px, tracking .14em, weight 700, var(--faint) (Login.tsx:50,58); Flutter AppTextField (app_text_field.dart:41-48) renders labels sentence-case sans 12.5px weight 600 AppColors.dim — wrong case, family, tracking, size, color. (3) PRIMARY BUTTON COLOR — web uses var(--primary)/var(--onPrimary); Flutter AppButton primary resolves to AppColors.accent (near-white), wrong in Light. (4) TYPOGRAPHY FAMILY — design guide (README.md:37) names Circular XX Web as primary family; AppFonts.sans is 'Hanken Grotesk' (tokens.dart:47). (5) INPUT FOCUS — web animates border to var(--text) on focus; Flutter uses sc.TextField default focus ring (app_text_field.dart:51-60), not the same border-color treatment. (6) FLUTTER-ONLY EXTRA — a top-right server-config gear/host button (login_screen.dart:191-198,213-229) + _configureServer dialog exist because the desktop client is backend-agnostic; web has no such control. Required desktop affordance — KEEP it, do not remove to match web.

- **Primary button color token** — Point AppButton primary at theme --primary/--onPrimary rather than AppColors.accent (near-white) so the Sign-in pill is correct in Light as well as Dark.
- **Field label typography** — Change AppTextField label rendering (app_text_field.dart:41-48) to UPPERCASE JetBrains Mono ~11.5px, letterSpacing .14em, weight 700, faint color — matching Login.tsx:50,58 — instead of sentence-case sans 12.5px w600 dim.  _(risk: Verify uppercasing does not double-apply if any caller passes already-uppercased labels)_

- **Server-config control (Flutter-only)** — Keep the top-right gear + host button and _configureServer dialog (login_screen.dart:191-229); restyle it to read theme tokens (currently AppColors.faint) so it stays legible in Light.
