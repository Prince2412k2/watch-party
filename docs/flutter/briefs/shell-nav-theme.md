# Recon brief: shell-nav-theme

**Task:** Rebuild Flutter desktop shell to match redesigned web: bottom-centered nav, top-right profile menu, bottom-right popcorn control, and a three-theme (Light/Balanced/Dark) system with artwork wash

**Effort:** default  **DependsOn:** ['design-tokens']

## Redesigned web (behavior source of truth)

The redesigned desktop shell is `WebShell` (app/client/src/components/WebShell.tsx:203-283), rendered by App.tsx:85-92 for four desktop routes: /library|/movies -> 'movies', /series -> 'series', /discover -> 'discover', /downloads -> 'downloads'. COMPOSITION (WebShell.tsx:258-282): a `.web-app` root carrying `data-theme={theme}` (styles.css:197-245), `position:fixed; inset:0; overflow:hidden; isolation:isolate` — edge-to-edge, NO top bar, NO sidebar, NO wordmark, NO rounded outer frame. THREE PERSISTED THEMES: state in WebShell `useState<Theme>` seeded from `localStorage['watchparty-theme']`, default 'light' (WebShell.tsx:211-214), written back on change (WebShell.tsx:222-224). light|dark|balanced applied purely via `data-theme` CSS-var swaps: base `.web-app` block (styles.css:197-226) IS the dark palette (--wp-bg #101113, --wp-text #f2f1ed, --wp-surface #17181b/#222327/#2c2e33, --wp-line rgba(255,255,255,.1/.17), --wp-stage rgba(15,16,18,.82), --wp-shadow rgba(0,0,0,.36)); `[data-theme=light]` (styles.css:227-241) swaps to #ffffff bg / #171719 text / #ebeae5 surface ramp / dark lines / stage rgba(255,255,255,.84); `[data-theme=balanced]` (styles.css:242-245) KEEPS the dark palette but lowers stage opacity to .76 so artwork dominates. Ambient opacity per theme: balanced 1, dark .42, light .3+brighten (styles.css:261-263). ARTWORK WASH is the `--balanced-poster` CSS var set on document.documentElement to `url(Backdrop), url(Primary)` of the focused/hovered/first poster (pages/Library.tsx:70-73, called Library.tsx:223 for first item and 609/776 on hover+focus), consumed by `.web-ambient` background-image with radial-gradient fallback (styles.css:250-252) and `filter:blur(72px) saturate(.9); transform:scale(1.12)` (styles.css:255-256), transitioning opacity+background-image over .35s.

## Current Flutter (divergence)

The Flutter client is built to the OLD design: dark-only, left nav rail, top window/section bar. It DIVERGES from the redesign on every axis of this surface.

ROOT (flutter_app/lib/app/app.dart:44-123): `MaterialApp.router` hardcodes a single theme — `theme/darkTheme: AppTheme.dark`, `themeMode: ThemeMode.dark` (app.dart:48-50), and `sc.ShadcnLayer(theme: AppShadcnTheme.dark, themeMode: sc.ThemeMode.dark)` (app.dart:116-119). No light/balanced, no theme state, no persistence.

TOP CHROME (against the guide): a frameless `VirtualWindowFrame` wraps everything and `_WindowBar` (app.dart:126-267, height 44) paints a full-width top strip with a draggable section-title region (`shellSectionTitle`, app_shell.dart:56-61), a Search/command-palette trigger, a Sign out button, and min/maximize/close window controls (Windows/Linux right, macOS traffic-lights left). Above the router it also stacks `GlobalPartyBar` (app.dart:63-73), a slim top party strip (ui/widgets/global_party_bar.dart:23-90) showing status + Return/End — a TOP bar, not a bottom-right popcorn control.

SHELL (flutter_app/lib/app/screens/app_shell.dart:72-154): a `Scaffold` whose body is `Row[ NavRail (left) , VerticalDivider , Expanded(content) ]` (app_shell.dart:100-108) plus a `Positioned(top16,right20)` profile/login button (app_shell.dart:109-116). `NavRail` (ui/widgets/nav_rail.dart:26-197) is a LEFT vertical rail (width 240, compact 72 under 720px) with a 'Watchparty' wordmark (nav_rail.dart:65-72) and SIX destinations from `kShellDestinations` (app_shell.dart:13-36): Home /home, Browse /browse, Party /party, Downloads /downloads, Offline /offline, Find /servarr. Active row = surface-2 pill fill + weight700 (nav_rail.dart:106-131). Routes (app/router.dart:30-39,113-172) are /home,/browse,/party,/downloads,/offline,/servarr inside a ShellRoute — NOT the web's /movies,/series,/discover,/downloads, and Party is a nav destination rather than a floating control.

PROFILE (app_shell.dart:184-220): `_ProfileButton` is a single 40px circle (person icon) that on tap goes STRAIGHT to a Sign-out confirm dialog (app_shell.dart:189-197). No dropdown, no 'Signed in as', no theme switch, no notification dot.

THEME TOKENS (flutter_app/lib/ui/tokens.dart:8-30): one monochrome dark set (bg #0A0A0B, text #F4F4F5, surfaces, red #E0655E) — values close to the web dark palette but NOT matching the redesign's #101113/#17181b ramp, and there is no light or balanced variant. Font family is 'Hanken Grotesk' (tokens.dart:48), NOT 'Circular XX'; fonts and popcorn.png are NOT declared in pubspec.yaml (pubspec ends at flutter: uses-material-design:true, lines 70-72). No ambient artwork-wash layer exists anywhere. `AppTheme` (theme.dart) and `AppShadcnTheme` (shadcn_theme.dart) each expose only a `.dark`. `Scrim` (ui/widgets/scrim.dart) is a dimming overlay, unrelated to the ambient wash. Motion tokens (ui/motion.dart, tokens.dart:58-79) exist and are reusable.

Net: the party session provider is preserved (non-autoDispose, survives nav — global_party_bar.dart:14-19), auth logout flow works, window is frameless — but the shell layout, nav placement/identity, profile menu, popcorn control, three-theme system, and artwork wash are all absent.

## Open questions (resolved by plan decisions where applicable)
- Route identity: should Flutter rename its routes to the web's /movies,/series,/discover,/downloads, or keep /home,/browse,... and only relabel the 4 tabs? This affects go_router guards, command palette, and shortcuts.
- The web has exactly 4 nav destinations; Flutter currently has 6 (adds Party, Offline, Find/Servarr). Where do Offline and Find/Servarr live after the redesign - folded into Discover/Downloads, into the profile menu, or dropped from primary nav?
- Desktop window controls: the guide forbids a top bar, but a frameless Flutter window needs drag/min/max/close. What presentation is acceptable (floating overlay controls vs a minimal invisible drag region)?
- Is the mobile/compact layout in scope for this surface, or desktop-only for now? Web has responsive tweaks (styles.css:588-596) but Flutter is desktop-first.
- Are the Circular XX font files licensed/available to bundle into the Flutter app (they exist under app/client for web)?
- Should the ambient artwork wash be global (updated from Library/Discover hover) or is a per-screen wash acceptable for v1, given the shared 'current artwork id' state does not yet exist in Flutter?
