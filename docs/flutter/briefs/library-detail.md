# Recon brief: library-detail

**Task:** Rebuild Flutter library shelves + movie/show detail to match redesigned web

**Effort:** default  **DependsOn:** ['design-tokens', 'shell']

## Redesigned web (behavior source of truth)

The redesigned web library lives entirely in app/client/src/pages/Library.tsx (1114 lines), one component driven by a nav `stack`. Three render modes:

HOME (no stack) — HomeView (Library.tsx:543-603). Ordered horizontal rails via the shared `Rail` component (Library.tsx:650-677): "Continue watching" (StillCard, 16:9, 300px, watch-progress bar), "Recently added" (PosterCard, 2:3, 170px, mono "NEW" badge — from GET /api/library/latest), "Downloading now" (DownloadingCard w/ progress ring, from /api/servarr/downloads/enriched, standalone only), "Next up" (StillCard 16:9), "Libraries" (ViewCard, 16:9 with gradient + icon+name overlay). Each Rail renders an h2 heading `fontSize:21 fontWeight:800` + a right-aligned count+chevron (Library.tsx:660-666), a horizontally-scrolling flex track (`gap:16, overflowX:auto, scrollSnapType:x proximity`, Library.tsx:669), and hover-reveal circular scroll arrows on the right (RailArrow, Library.tsx:678-689). Data from GET /api/library/home → {views, resume, nextUp} (Library.tsx:178).

LIBRARY/SEASON GRID (stack, non-detail) — GridView (Library.tsx:1085-1113) over GET /api/library/items/:id/children. Despite the name it renders HORIZONTAL SHELVES: `PosterWall` (Library.tsx:750-762) declares a grid inline but styles.css:512 forces `display:flex !important; overflow-x:auto` — a scroll rail, not a grid. Heading is a large light title (`.library-title-row h1` clamp 38-44px, weight 300, styles.css:502). Cards are `PosterCardFluid`/`WallPoster` (Library.tsx:764-798): 3/5 aspect art (styles.css:517), episode-count / watched-check / season badges, and a CENTERED caption block below with title + a 5-star rating row (`.library-poster-meta`/`-title`/`-stars`, styles.css:519-522, text-align:center). The FIRST card is subtly enlarged `transform:scale(1.035)` (styles.css:515); hover only deepens shadow, never translates (styles.css:518). Additional genre-subset shelves are appended when a genre is a meaningful subset (Library.tsx:1087-1089,1104-1109). No search field.

DETAIL — `Details` (Library.tsx:801-962) over GET /api/library/item/:id (+ /playback-info for tracks, +children for series seasons/episodes). Fullscreen cinematic stage (`.library-detail-stage` 100% height min 620px, styles.css:530): absolute Backdrop image (objectPosition center 28%, saturate/contrast filter) + a theme-aware two-axis `.library-detail-wash` (horizontal left-dark→right-light PLUS vertical bottom-dark, styles.css:533-536). Circular glass Back button top-left (styles.css:537). Content is a 2-col grid (styles.css:538, `.92fr / 1.08fr`, vertically centered): LEFT `.library-detail-copy` = genres as "A  /  B  /  C" mono-uppercase text (Library.tsx:918, styles.css:540), h1 title (clamp 40-56px, weight 300, styles.css:541), optional episode label, 4-line-clamp synopsis (styles.css:542), a mono-uppercase meta line with dot separators (★rating, officialRating, runtime, premiere, res/HDR/size — Library.tsx:922-926, styles.css:544-546), then actions (Library.tsx:927-934): a pill Watch button whose label is `Resume {label}` / `Play first episode` / `Watch now` (Library.tsx:930), plus (movies only) a circular track-menu button opening `DetailTrackMenu` (audio + subtitle selection, SRT/VTT upload+delete, Library.tsx:1016-1083). RIGHT column = for movies a 2/3 poster (styles.css:551); for series a right-aligned `.library-seasons` selector list with animated underline indicators (Library.tsx:941-949, styles.css:552-556). BOTTOM = for movies an absolutely-positioned cast strip of 44px circular portraits with name+role (StagePerson, Library.tsx:964-975, styles.css:557-562); for series an `.library-episodes-dock` horizontal episode rail (Library.tsx:956-958, EpisodeRow/EpisodeCard 977-1009, styles.css:563-571). Selecting a season/episode updates `activeId` IN PLACE (no navigation) and re-fetches playback + drives party.shareView (Library.tsx:818-826). Watch calls `onWatch(playItem, {audioStreamIndex, subtitleStreamIndex})` (Library.tsx:928,901) so track selection flows into party/playback. Runtime formatting from fmtRuntimeFromTicks (format.ts:36). Mobile (styles.css:607-621) makes the stage scrollable, stacks the columns, and pushes the poster to a faint full-bleed background.

## Current Flutter (divergence)

The Flutter client splits these into three screens and was built to the OLD design.

HOME — flutter_app/lib/app/screens/home_screen.dart. A vertical ListView of `_Rail`s (home_screen.dart:137-201): Recently Added, Continue Watching, Next Up, Libraries — ALL rendered as identical 2:3 `PosterCard`s (home_screen.dart:179-192). No 16:9 StillCards for resume/next-up, no "NEW" badge, no 16:9 ViewCard library tiles, no "Downloading now" rail. Rail order differs from web (latest first vs web's resume first). Heading via `SectionHeader` (section_header.dart) at 16px/w700 — far from web's 21px/w800 with count+chevron. No hover scroll arrows. Page title is a plain 'Home' Text (home_screen.dart:102).

LIBRARY BROWSE — flutter_app/lib/app/screens/browse_screen.dart. A `GridView.builder` responsive POSTER GRID (browse_screen.dart:127-160, `SliverGridDelegateWithFixedCrossAxisCount`, childAspectRatio 0.52) — exactly the grid the redesign forbids. Has a search `AppTextField` (browse_screen.dart:63-67) and a shadcn segmented type filter (browse_screen.dart:72-82) — the redesign removes the search field. No genre-subset shelves, no enlarged first poster, no centered captions/star rows.

POSTER CARD — flutter_app/lib/ui/widgets/poster_card.dart. 2/3 aspect (web wall uses 3/5), sc.Card frame, hover AnimatedScale 1.03 (no upward translate — compliant), a LEFT-aligned title + faint subtitle below (poster_card.dart:112-128). Web wants the caption CENTERED with a star-rating row beneath the title. No enlarged-selected variant, no episode-count/watched/season badges.

DETAIL — flutter_app/lib/app/screens/detail_screen.dart. `_DetailBody` (detail_screen.dart:127-163) is a SingleChildScrollView: a 520px-tall `_Hero` backdrop Stack at the TOP with a single bottom→top gradient (detail_screen.dart:208-241) — NOT a fullscreen stage, NOT the two-axis theme-aware wash. Single-column hero copy: a logo-or-text title via `_TitleOrLogo` (detail_screen.dart:345-365) — but the web redesign dropped the logo and uses a plain weight-300 h1; rating/genres rendered as `AppChip`s (detail_screen.dart:262-279) instead of web's "/"-joined genre text + dotted mono meta line; 3-line overview; an action Row with Play/Resume/'Browse episodes', a Flutter-only `DownloadButton`, and a subtitles `IconButton` opening a separate `showSubtitleManagerDialog` (detail_screen.dart:300-331). NO right-side movie poster. Series: `_ChildrenSection` (detail_screen.dart:396-449) shows Seasons as a horizontal PosterCard rail and the Play button is 'Browse episodes' with `onPressed:null` (dead) — tapping a season NAVIGATES to /detail/:seasonId; there is no in-place season selector + episode dock, no in-place activeId swap. Cast `_CastSection` (detail_screen.dart:520-592) is a below-the-fold horizontal rail of 96px circular portraits (web uses a 44px bottom strip). An extra `_LinksSection` (IMDb/TMDb pills, detail_screen.dart:594-634) has no web counterpart. Track selection does NOT flow into playback: `_playerRoute` (detail_screen.dart:652) carries only itemId/title, no audioStreamIndex/subtitleStreamIndex. Data wiring is correct via library_provider.dart (homeProvider, latestProvider, itemDetailProvider, itemChildrenProvider, browseItemsProvider). gallery_screen.dart is a dev-only widget showcase, not user-facing.

## Gaps to close

- **Library browse is a grid, not shelves** — Replace browse_screen.dart's GridView.builder with vertically-stacked horizontal shelves. Each shelf = a large light heading (mirror web .library-title-row: ~40px weight 300) with right-aligned scroll arrows, over a horizontal scroll rail of poster cards (gap 16, scroll-snap). Remove the responsive grid entirely.  _(risk: Preserve browseItemsProvider + type filter behavior and the poster→detail Hero tag (poster-<id>) uniqueness; do not break onTap navigation to /detail/:id.)_
- **Remove search field from library/discover** — Per design guide (no search field in library/discover), drop the AppTextField from browse_screen.dart:63-67 and the browseQueryProvider dependency for this surface; keep the Movies/Shows separation via type rails.  _(risk: browseQueryProvider/search API is still used elsewhere; only remove its UI on this surface, do not delete the provider.)_
- **Poster card caption + star rating + aspect** — Update poster_card.dart: center the title caption below the artwork and add a 5-star community-rating row beneath it (web .library-poster-meta/-title/-stars). Support a 3/5 aspect for shelf cards (web wall) while keeping 2/3 for rails. Add optional badges (episode-count, watched-check, season 'S#'). Keep hover shadow-only (no translate) — already compliant.  _(risk: PosterCard is a FROZEN-contract widget used by home/browse/detail-season rails; keep existing params additive and default-compatible so other call sites don't regress.)_
- **Enlarged first/selected poster in shelves** — Scale the first poster in each shelf to ~1.035 with left-bottom transform origin, and ensure rail padding contains the scale+shadow with no clipped tops (web styles.css:515, guide lines 88-90).
- **Genre-subset shelves** — After the main shelf inside a library, append extra shelves for genres that form a meaningful subset (web Library.tsx:1087-1089): more than one item and fewer than the whole set; never duplicate the full shelf.
- **Detail is not a fullscreen cinematic stage** — Rebuild detail_screen.dart _DetailBody into a fullscreen stage: full-bleed backdrop filling the viewport (objectPosition ~center 28%) under a two-axis theme-aware wash (horizontal left-dark→right + vertical bottom-dark), a circular glass Back button top-left, and a 2-column layout (~.92fr left copy / 1.08fr right) vertically centered — not a 520px hero over a scrolling column. Make it non-scrolling on desktop; scrollable/stacked on narrow widths.  _(risk: Must not remount playback/party systems; keep DownloadButton (offline) and Start-Party affordance wired.)_

- **Detail right column: movie poster / series season selector** — For movies, add the right-side 2/3 primary poster (web .library-detail-poster). For series, replace the season POSTER RAIL + navigation with a right-aligned season-selector list (animated underline indicators) that swaps the active season IN PLACE.  _(risk: Series season/episode selection must update the playback target and party.shareView state in place (web Library.tsx:818-826); do not navigate to a new route per season.)_
- **Series episode dock + in-place episode selection** — Add a bottom horizontal episode rail (EpisodeCard: 16:9 thumb, E# badge, progress, selected outline). Selecting an episode updates activeId/metadata and the Watch target without leaving the screen. Series primary action label = 'Play first episode' when nothing selected (web Library.tsx:930), not a dead 'Browse episodes'.  _(risk: Episode selection feeds the playback item id + track selection into the player/party; keep that data flow intact.)_
- **Cast strip placement/size** — Move cast to a restrained bottom strip of ~44px circular portraits (name + role, initials fallback) per web StagePerson, rather than the 96px below-the-fold rail.
- **Track selection must flow into playback** — Add an inline audio+subtitle track menu (circular button beside Watch) that reads /playback-info and passes selected audioStreamIndex/subtitleStreamIndex into the player route (and into party.selectMedia) — web Library.tsx:901,928. - **Remove IMDb/TMDb links section** — Drop _LinksSection (detail_screen.dart:594-634) — the redesigned web detail has no external-links block.
