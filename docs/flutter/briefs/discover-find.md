# Recon brief: discover-find

**Task:** Rebuild Flutter Discover as a search-free two-rail browse + full acquire flow

**Effort:** default  **DependsOn:** ['design-tokens', 'shell', 'library-shelves']

## Redesigned web (behavior source of truth)

The redesign COLLAPSES "browse" and "acquire" into ONE page. `app/client/src/pages/FindDownload.tsx` exports `Browse`, mounted at `/discover` (App.tsx:91) as the "Discover" nav destination (Movies, Shows, Discover, Downloads — README:79). There is NO longer a separate Find page; Discover IS the servarr search/grab surface.

DISCOVER (no search field — README:94). Main render (FindDownload.tsx:357-372) shows a `.discover-rows` scroll column with exactly TWO stacked rails via `PopularRail` (l.361-368): one kind="movie" (radarr), one kind="series" (sonarr). Each rail fetches `GET /api/servarr/{service}/discover?page=1` (loadDiscover, l.180-193; server route index.js:568/860, returns `{source, items}`). Heading label is source-driven: `source==='tmdb_trending'` → "Trending this week", else "Discover", plus a mono kind label "Movies"/"Shows" (l.427-432). CSS `.discover-rows h2` forces font-weight 300, clamp(38-44px), letter-spacing -.04em (styles.css:577) — large thin headings. Rail is a horizontal-scroll flex row (ResultGrid `rail`, l.521-522); each card is aspect 2/3, width clamp(150-205px). NOTE: `SearchBar`, `SuggestedSearches`, `NotAvailable` components still exist in the file but are DEAD (never rendered) — leftovers from the old search-first design.

CARD (`ResultCard`, l.533-659): poster (remoteUrl→url fallback, l.67-71), rating badge top-left (l.557), hover reveal with overview + a state-driven control. State comes from a per-item request machine keyed by tmdb/tvdbId (l.219, keyOf l.99): idle→Download button; `searching`→"Finding a release…"; `grabbed`/live-torrent→"Downloading · N%" + progress bar; `monitoring`→"Added — monitoring"; `no_release`/`search_failed`→red + retry; `added`→"In library" + check. `outcomeToState` (l.109-118) maps server outcomes. Live % overrides state via `matchTorrent` (normalized-title substring, l.137-141) against the shared `useTorrents` poller.

LIVE DOWNLOADS (`useTorrents.ts`): visibility-aware 2.5s poll of `GET /api/servarr/downloads/enriched`, shared by cards+detail (FindDownload.tsx:226). Exposes pause/resume/remove; `isPausedState`/`isActiveState` classify qBittorrent state strings.

DETAIL (`DetailView`, l.662-897): FULL-PAGE (not a sheet) — blurred backdrop hero (backdropUrl fanart→poster, l.73-77), two gradient washes, 190px poster, MOVIE/SERIES eyebrow, 40px title, rating+genres row, mono info line (year/runtime/cert/seasons/network/status). A floating back button (l.339-344). Primary action branches by kind:
- MOVIE: one-tap Download → `oneTapAdd` POSTs `/{service}/request` with meta defaults (l.301-318); gear → `OptionsDialog`; "See all sources"/"Choose a release" → `ReleasePicker`; "Add source" → `ManualSourceDialog`; plus live-download/searching/monitoring/noRelease/searchFailed/added states; Remove-from-library with confirm modal (l.350-354, DELETE `/{service}/movie|series/{id}?deleteFiles=true`).
- SERIES: `SeasonChooser` (l.907-1024) — per-season rows (SeasonRow l.1028-1090) with idle/requesting/requested/monitored/error states, "All seasons" button, Specials section; POSTs `/sonarr/request-season` {series, seasons[], profiles}.

DIALOGS: `OptionsDialog` (l.1269-1416) — quality-profile + root-folder selects, series language + monitor/searchNow toggles, POST `/{service}/request`, outcome→Notice. `ReleasePicker` (l.1429-1636, movies only) — POST `/radarr/releases` (adds monitored+no-search if not present), lists releases with quality/seeders/leechers/size/indexer/rejections (ReleaseRow l.1583), grab via POST `/radarr/grab`, and a createdByPicker cleanup lifecycle that POSTs `/radarr/releases/cancel` on every close/unmount path exactly once (l.1440-1497). `ManualSourceDialog` (l.1095-1266) — magnet/.torrent tabs, resolveTargetId add-then-lookup (l.1111-1138), POST `/manual/magnet` (JSON) or `/manual/torrent` (x-bittorrent body, 2MiB cap), series season/episode coords. Metadata (profiles/rootFolders/langProfiles) session-cached per service (loadMeta l.146-160). `useFailingDownloads.ts` polls radarr+sonarr queues for the failing count (nav badge / Downloads page), not used on Discover itself.

## Current Flutter (divergence)

Two screens map here. `flutter_app/lib/app/screens/servarr_screen.dart` (`ServarrScreen`, titled "Find & Download", servarr_screen.dart:42) is built to the OLD design: a `SectionHeader` + a `_SearchBar` (l.44-51/104-173) with a Movies/Series toggle AND a text field, then health-gated content that shows either search results or a single popular rail. Results and popular render as a `GridView` (`_ResultGrid` l.240-280, `_GridSkeleton` l.175-202, maxCrossAxisExtent 180, aspect 0.5) — a responsive poster GRID, not a shelf. Popular uses `_PopularRail` (l.204-238) which shows ONLY the currently-selected kind (`servarrPopularProvider(kind)`), never two rails.

State: `servarr_provider.dart`. `ServarrSearchNotifier` (l.204-322) implements debounced search, kind toggle, and the one-tap `request` flow (l.275-303) → POST `{service}/request` with meta defaults, `_outcomeToState` (l.92-99) mirrors the web machine; `ServarrRequestState` enum (l.81-90) is complete. `_StatusControl` (servarr_screen.dart:441-509) and `_DetailAction` (l.676-731) render every request state. `servarrPopularProvider` (l.153-159) calls `{service}/popular` (NOT `/discover`). `servarrMetaProvider` caches profiles/rootFolders/langProfiles (l.133-150). Health via `servarrHealthProvider`/`servarrServiceReady` (l.102-117). `ServarrTitle` wrapper (l.25-78) already parses posterUrl/rating/genres/seasons/isAdded/key.

Detail: `_showDetail` opens a `showModalBottomSheet` (`_DetailSheet` l.557-674) — a bottom sheet with poster+meta+genres+overview and a SINGLE `_DetailAction` button. No blurred-backdrop full-page hero, no gear/Options, no release picker, no season chooser, no manual source, no remove.

Live downloads: `servarrDownloadsPollProvider` (l.364-392, 4s poll of `downloads/enriched`) and `servarrFailingQueueProvider` (l.411-442) exist but are consumed ONLY by `servarr_queue_screen.dart` (the "Downloads" destination). Cards/detail on Discover do NOT subscribe to it, so there is no live "Downloading · N%" overlay and no torrent-title matching.

`servarr_queue_screen.dart` (`ServarrQueueScreen`, the Downloads destination) is close to functional parity: "Needs attention" (failing queue, remove) + "Active" downloads (MediaRow with progress, pause/resume/delete). ApiClient servarr passthroughs (api_client.dart:402-433) cover GET/POST/DELETE + image proxy. NOTE: `servarrDelete` only forwards query params, so queue `blocklist`-on-delete isn't reachable (flagged servarr_provider.dart:461-466).

Divergence from redesign+guide: (1) a search field exists — guide forbids it; (2) grid instead of horizontal shelves; (3) one kind-selectable rail instead of two fixed Movies+Shows rails; (4) wrong endpoint (`/popular` vs `/discover`); (5) bottom-sheet detail with one button instead of full-page hero + full acquire flow; (6) no live download progress on cards/detail; (7) missing OptionsDialog, ReleasePicker, SeasonChooser, ManualSourceDialog, remove-from-library.

## Implementation brief

- **Approach:** 1) Strip search from Discover: remove _SearchBar and the search-results branch in ServarrScreen; keep ServarrSearchNotifier's request/stateFor/remove/requestStates machine and _outcomeToState, delete term/debounce/_search/results/hasSearched. Render a scroll column of two horizontal shelves (Movies=radarr, Shows=sonarr) reusing the Library shelf/PosterCard widget for poster+centered-title styling; overlay the rating badge and the live status control the servarr cards need. 2) Point discover at the right feed: rewrite servarrPopularProvider to GET `{service}/discover?page=1`, parse `{source, items}`, expose source for the heading ('Trending this week' vs 'Discover'); per-kind family so each rail loads/degrades independently. 3) Promote the downloads poller to page scope (visibility-aware ~2.5s, one shared poll) and port matchTorrent + isPausedState/isActiveState; feed live torrent state into cards and detail so 'Downloading · N%' overrides request state. 4) Replace the bottom-sheet detail with a pushed full-page DetailView: blurred backdrop (fanart→poster), poster, MOVIE/SERIES eyebrow, title, rating+genres, mono info line, overview, floating Back. 5) Movie detail wires Download(one-tap request) + gear→OptionsDialog + See-all-sources→ReleasePicker + Add-source→ManualSourceDialog + Remove(confirm→DELETE deleteFiles). 6) Series detail wires SeasonChooser→POST `/sonarr/request-season`. 7) Port ReleasePicker's createdByPicker cancel lifecycle faithfully (cancel-once on every dismiss AND on dispose, never after grab). Match the endpoints already used by web verbatim; all reachable through existing ApiClient.servarrGet/Post/Delete passthroughs.
- **Files to edit:** flutter_app/lib/app/screens/servarr_screen.dart, flutter_app/lib/state/servarr_provider.dart
- **Files to create:** flutter_app/lib/app/screens/servarr_detail_screen.dart, flutter_app/lib/app/screens/servarr_options_dialog.dart, flutter_app/lib/app/screens/servarr_release_picker.dart, flutter_app/lib/app/screens/servarr_season_chooser.dart, flutter_app/lib/app/screens/servarr_manual_source_dialog.dart
- **Preserve (must not break):**
  - ServarrRequestState machine + _outcomeToState → server outcome mapping must stay intact
  - ReleasePicker must POST /radarr/releases/cancel exactly once for entries it created, on any close/unmount, and never after a successful grab (no orphaned Radarr entries)
  - Manual .torrent upload must send raw file body as application/x-bittorrent with the 2MiB cap; magnet via /manual/magnet JSON
  - request()/request-season POST bodies must keep the exact shape (movie/series raw JSON + profileId + rootFolderPath [+ languageProfileId/monitor/searchNow])
  - Live-torrent 'downloading' must take precedence over searching/monitoring states, matched by normalized title
  - Not-yet-added series must not display seasons as already-monitored unless isAdded
  - Downloads/queue screen (servarr_queue_screen.dart) and its pollers must keep working (shared poller change must not break it)
