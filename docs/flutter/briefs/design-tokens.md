# Recon brief: design-tokens

**Task:** Rebuild the Flutter design system as a three-theme (Light/Balanced/Dark) token + ThemeData layer matching the redesigned web --wp-* system, with Circular XX fonts, brand red, and the balanced artwork-wash

**Effort:** default  **DependsOn:** []

## Redesigned web (behavior source of truth)

The redesigned web design system lives in two layers of app/client/src/styles.css.

LAYER A — legacy `:root` block (styles.css:32-71): the OLD "cinematic-minimal, dark-only, monochrome" tokens (--bg #0a0a0b, --glass #141416, --accent near-white #F4F4F5, one semantic red #E0655E). This is exactly what the current Flutter tokens were ported from. It is NOT the redesign target for the main shell — it survives only as the fallback default for the download surfaces' inline styles.

LAYER B — the redesign, the real source of truth: the `.web-app` shell (styles.css:197-263) redefines everything through `--wp-*` custom properties and supports THREE persisted themes via `data-theme` on the root `.web-app` div (WebShell.tsx:259). The `--wp-*` block also re-aliases the Layer-A tokens (--bg, --text, --glass, etc.) so even the download/Discover surfaces styled via lib/ui.tsx `C` (which read `var(--wp-bg,…)`, ui.tsx:16-37) inherit whichever theme is active through the CSS cascade. So all surfaces are theme-aware in the redesign.

## Current Flutter (divergence)

The Flutter design system is three files under flutter_app/lib/ui/ plus a barrel (ui.dart), all built for the OLD Layer-A monochrome design and DARK-ONLY. Nothing supports the three-theme model or brand red, and the type scale violates the redesign guide.

tokens.dart (flutter_app/lib/ui/tokens.dart:8-30): `AppColors` is a single set of `const Color` values ported 1:1 from the legacy `:root`/ui.tsx `C` block — bg #0A0A0B, surface #141416, surface2 #1E1E21, surface3 #2A2A2E, text #F4F4F5, dim .62, faint .36, line .08, line2 .14, accent near-white, red/live #E0655E, green #5AB98A. These are the LAYER-A values, NOT the redesign's `--wp-*` DARK values (bg should be #101113, surface #17181b, surface2 #222327, surface3 #2c2e33, text #f2f1ed, line .1/.17). There is no brand red #d81f2a, no light/balanced palettes, no stage/nav/shadow tokens, no ambient-wash concept. AppSpacing radii (radiusSm 8, radius 10, radiusLg 16) don't match web (12/16 root, 22 for cards). AppFonts.sans is 'Hanken Grotesk' (tokens.dart:48) — wrong family; must be 'Circular XX'. AppMotion/AppBlur/AppElevation are usable but AppElevation shadows are ad-hoc, not the glass.tsx elevation scale.

theme.dart (flutter_app/lib/ui/theme.dart): exposes only `AppTheme.dark` (ColorScheme.dark, theme.dart:9-32) — no light or balanced ThemeData, and it's a static getter, not driven by a persisted mode. The type scale (theme.dart:67-117) is the biggest divergence from the guide: displaySmall is 28px w800 and titleLarge 20px w700 — the guide (README:47) explicitly rejects "extra-bold, oversized hero typography"; redesign headings are Circular Light w300 at 40-56px with -0.04em tracking. body is w500 (should be Book w400), and there is NO large display style for the 40-56px detail titles. letterSpacing values (-0.6, -0.4) are fixed px, not the -0.04em size-relative tracking the web uses.

shadcn_theme.dart (flutter_app/lib/ui/shadcn_theme.dart): one `AppShadcnTheme.dark` sc.ColorScheme mapping the legacy tokens; typography uses 'Hanken Grotesk'/'JetBrains Mono'. Single-theme, wrong palette, wrong font.

FONTS: no fonts are bundled — pubspec.yaml has `flutter: uses-material-design: true` with NO `fonts:` section, and there is no flutter_app/assets/fonts dir. The app currently relies on the 'Hanken Grotesk'→system fallback. The Circular XX files exist only as .woff2 (app/client/public/fonts/), which Flutter does NOT load — they must be converted to .ttf/.otf and declared with weight mappings.

DIVERGENCE SUMMARY: (1) dark-only vs three persisted themes; (2) legacy Layer-A palette vs redesign --wp-* palette in all three themes; (3) missing brand red #d81f2a; (4) missing balanced artwork-wash; (5) wrong font family + no bundled Circular XX; (6) guide-violating w800/28px type scale vs Circular-Light 40-56px display; (7) mismatched radii/shadow tokens.

## Gaps to close

- **Three-theme model (Light/Balanced/Dark)** — Replace the single dark-only AppColors/AppTheme with a theme-aware model exposing all three palettes with the EXACT --wp-* values (Dark: bg #101113 surface #17181b s2 #222327 s3 #2c2e33 text #f2f1ed dim .64 faint .38 line .1 line2 .17; Light: bg #ffffff surface #ebeae5 s2 #dfded8 s3 #d3d2cb text #171719 dim/faint on #171719 line .1/.18; Balanced: = Dark tokens with page bg #15161a and stage rgba(15,16,18,.76)). Drive it from a persisted enum (key equivalent to 'watchparty-theme', default light) surfaced in the profile menu, not a toolbar. Build separate ThemeData/sc.ThemeData per mode and switch at the app root.  _(risk: Theme must be a top-level rebuild that does NOT remount playback/party/socket/LiveKit widget subtrees (guide:141-152) — thread theme via an InheritedWidget/Riverpod provider above the shell but keep player/party state above or beside the theme boundary so switching never tears down sync.)_
- **Balanced artwork-wash** — Add a full-bleed blurred backdrop layer behind the shell that, in Balanced mode only, renders the selected title's Backdrop image (fall back to Primary, then to a radial gradient #47505a→#17181b) blurred ~72px, saturate .9, scaled 1.12, under a darkening gradient, at opacity 1; in Dark render the same layer at opacity .42; in Light at opacity .3 with stronger blur/brightness. The 'selected title' updates on library selection (mirrors Library.tsx:72-74 setting --balanced-poster). Expose a provider that the library/detail screens set.  _(risk: Image fetches must reuse the existing authed Jellyfin image pipeline (authed_image widget / library image API), not a raw NetworkImage, or auth breaks. Wash is purely decorative — must never block or gate content.)_
- **Circular XX fonts** — Convert CircularXXWeb-{Light,Book,Bold}.woff2 (app/client/public/fonts/) to .ttf/.otf, place under flutter_app/assets/fonts/, and declare family 'Circular XX' in pubspec.yaml with weight mappings: Light→w300, Book→w400 (also serve w500/w600 from Book), Bold→w700 (also w800/w900 from Bold). Compute letterSpacing as -0.04*fontSize (em→logical px) for headings.  _(risk: None functional — pure visual. But down-weighting headings from w800 to w300 must be applied everywhere headings are used or the app looks half-migrated.)_
- **Brand red #d81f2a** — Add a distinct brandRed token (#d81f2a) separate from semantic red (#E0655E). Keep semantic red for danger/destructive and live/download; add party-live green #78c99f alongside success green #5AB98A.  _(risk: Do not use brand red for destructive actions or vice-versa — the two reds carry different meaning; swapping them mislabels danger states.)_

## Implementation brief

- **Approach:** 1) Fonts: convert the three CircularXXWeb .woff2 (app/client/public/fonts/) to .ttf/.otf into flutter_app/assets/fonts/; add a `fonts:` section to pubspec declaring family 'Circular XX' (Light→w300, Book→w400/500/600, Bold→w700/800/900) and 'JetBrains Mono'. Set AppFonts.sans='Circular XX'. 2) Palette: define a WpPalette data class holding every semantic token (bg, surface, surface2, surface3, text, dim, faint, line, line2, stage, nav, shadow, accent, onAccent) and instantiate three consts with the EXACT hex/alpha values from webSummary (Dark/Light/Balanced). Add global brandRed #d81f2a, semantic red #E0655E, green #5AB98A, partyLive #78c99f. 3) theme_mode: an enum {light,balanced,dark} persisted via shared_preferences under 'watchparty-theme' (default light), exposed through a Riverpod provider; profile menu drives it. 4) theme.dart: make AppTheme build a ThemeData from a given WpPalette (not a static dark getter); rebuild the type scale to the Circular Light w300 40-56px / body Book w400 16-18px / label 14px scale with -0.04em (computed as -0.04*size) tracking on headings; remove the w800/28px styles. Provide displayLarge(detail 40-56), headlineLarge(section 34-44), titleLarge(20), body(16-18 lh1.75), label(14), posterTitle(14 w400), mono. 5) shadcn_theme: build one sc.ThemeData per palette, radius=12/12, flat surfaces (drop acrylic blur), correct fonts. 6) ambient_wash: a widget behind the shell that reads the selected-title image via the existing authed image pipeline and renders the blurred/gradient wash at opacity 1/.42/.3 for balanced/dark/light; expose a provider the library sets on selection. 7) Wire the chosen ThemeData + sc.ThemeData + palette InheritedWidget at the app root, ABOVE the shell but without remounting player/party/socket subtrees. 8) Update ui.dart barrel exports.
- **Files to edit:** flutter_app/lib/ui/tokens.dart, flutter_app/lib/ui/theme.dart, flutter_app/lib/ui/shadcn_theme.dart, flutter_app/lib/ui/ui.dart, flutter_app/pubspec.yaml
- **Files to create:** flutter_app/lib/ui/theme_mode.dart (persisted theme enum + provider, key 'watchparty-theme', default light), flutter_app/lib/ui/palette.dart (the three --wp-* palettes as data), flutter_app/lib/ui/widgets/ambient_wash.dart (balanced artwork backdrop layer), flutter_app/assets/fonts/CircularXX-Light.ttf, flutter_app/assets/fonts/CircularXX-Book.ttf, flutter_app/assets/fonts/CircularXX-Bold.ttf
- **Preserve (must not break):**
  - Theme switching must NOT remount or reset playback, watch-party sync, socket, LiveKit A/V, chat, or download state — the theme boundary sits above the shell chrome, not above the functional providers.
  - Balanced-wash images must use the existing authed Jellyfin image API/authed_image widget, never an unauthenticated fetch.
  - Persistence key and default (default light) match the web so a user's choice is consistent across clients.
  - Brand red (#d81f2a) and semantic red (#E0655E) stay distinct; destructive/live semantics keep the semantic red.

## Open questions (resolved by plan decisions where applicable)
- Font licensing/format: are we permitted to convert & bundle the Circular XX woff2 as ttf/otf in the desktop app, and is a suitable conversion available, or should the desktop client ship a licensed fallback family?
- Web persists theme in browser localStorage per-device; should the Flutter desktop client persist locally via shared_preferences (assumed) or sync the choice to the server profile so it follows the user across web+desktop?
- Balanced 'selected title' source: on web it's whatever title the library last focused; should the desktop wash follow library focus/hover, the detail screen, or the currently-playing title when in a party?
