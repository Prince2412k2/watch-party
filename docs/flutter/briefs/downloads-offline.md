# Recon brief: downloads-offline

**Task:** Bring the Flutter servarr Downloads page + offline library to 1:1 with the redesigned web app (poster-grid rings, DownloadDetail overlay, needs-attention danger cards, delete-with-files dialog)

**Effort:** default  **DependsOn:** ['design-tokens', 'shell']

## Redesigned web (behavior source of truth)

The redesigned web app has TWO distinct download systems, both in this surface:

1) SERVER-SIDE SERVARR/qBittorrent downloads = the "Downloads" nav destination (design README:79; implemented/downloads.png shows this page's "Downloads unavailable" fallback). Source: app/client/src/pages/Downloads.tsx + DownloadDetail.tsx + hooks/useTorrents.ts.
- Layout: full-viewport scroll, padding 64px 42px 100px, maxWidth 1600 centered (Downloads.tsx:118-129). No page title; two sections.
- NeedsAttention section (Downloads.tsx:136-214): Radarr/Sonarr queue items stuck between grab and import. Heading "Needs attention" (DANGER red #E0655E when items>0) + mono "N stuck" + reconnecting spinner. Each FailingRow is a danger-bordered flat card (#141416, DANGER_BORDER): alert-icon tile, title, per-reason lines from statusMessages/errorMessage (red), mono "indexer · size" line, Remove button that expands an INLINE confirm row with Cancel / Remove / "Remove & block" (blocklist=true) (Downloads.tsx:194-211).
- ActiveDownloads section (Downloads.tsx:226-292): heading "Active" + mono aggregate "N active · ↓ {fmtSpeed(dl)} · ↑ {fmtSpeed(up)}" (:243-250) + reconnecting spinner. Body is a POSTER GRID (repeat(auto-fill, minmax(160px,1fr)), gap 18) of TorrentCard (:306-338): 2:3 DownloadPoster with a circular DownloadRing (%) overlaid on a black scrim, a pulsing red "DL" live dot top-left while active, title + mono "{stateLabel} · {subtitle}" line where subtitle defaults to "↓ speed", and an inline pause/resume + delete icon-button row. Clicking the poster opens the DownloadDetail overlay.
- DownloadDetail full-screen overlay (DownloadDetail.tsx:143-291): fetches GET /api/servarr/downloads/:hash/detail (:129) for rich metadata; blurred-poster hero backdrop with two black scrims, back chevron, poster+ring, a "DOWNLOADING/PAUSED/FINISHING UP" pill with pulse dot, big title (40px), subtitle, rating(star)+genres, mono info line (year·runtime·cert·network·status), a large DownloadRing (104px) + a mono stats grid (↓ speed / ETA / Seeds / Peers / size), Pause/Resume pill (accent) + Delete pill (danger), overview paragraph. Delete opens DeleteConfirm modal with an "Also delete downloaded files" toggle.
- DeleteDialog (Downloads.tsx:375-408): modal on black scrim, danger trash icon, title "Remove download?", "{name} will stop downloading", "Also delete downloaded files" toggle, Cancel / Remove(danger).
- Empty/unavailable states: "No downloads", "Downloads unavailable"/"Downloads not configured" (:410-418).
- Data/behavior: useTorrents(ready) polls GET /api/servarr/downloads/enriched every 2.5s, visibility-aware, single AbortController, keeps last-good on error and flags reconnect (useTorrents.ts:47-99). activeCount uses DOWNLOADING_STATES set; isPausedState covers paused*+stopped* (qBit 5.x). Actions POST /api/servarr/qbittorrent/{pause|resume|delete} with optimistic update + re-poll. NeedsAttention polls /api/servarr/radarr/queue + /sonarr/queue every 6s, filters failing, DELETE /api/servarr/{service}/queue/{id} with {blocklist} body (Downloads.tsx:49-95). Colors: monochrome — near-white progress, single DANGER red for error/live/destructive only.

2) NATIVE ON-DEVICE offline downloads (native-only, IS_NATIVE gated). Source: native/offline/OfflineLibrary.tsx, DownloadButton.tsx, DownloadProgress.tsx.
- DownloadProgress row (DownloadProgress.tsx): flat card (C.surface, C.line), active red dot, title (red if error), a THIN 4px progress bar (accent fill, C.faint when paused), mono line "{stateLabel} · received/total · pct · speed", pause/resume/cancel 32px icon buttons, green check when done.
- OfflineLibrary (OfflineLibrary.tsx): two sections — "Downloading" (list of DownloadProgress rows for state!=='done') and "Available offline" (flat rows: title + formatBytes(sizeBytes) + trash button). Empty = Notice "Nothing downloaded yet". This is a FLAT vertical list, NOT a poster grid.
- DownloadButton (DownloadButton.tsx): per-title states — "Download" / progress pill ("Queued"/"Paused"/"pct" + received/total + pause+cancel) / "Retry" on error / "Downloaded" chip (green check) + trash. Uses useDownloads/useOfflineLibrary.

## Current Flutter (divergence)

Flutter splits these systems across three routes reached from a LEFT NAV RAIL (app_shell.dart:13-36 — Home/Browse/Party/Downloads/Offline/Find; the rail itself is old-design shell, out of this surface's scope):

SERVARR side — flutter_app/lib/app/screens/servarr_queue_screen.dart (route /servarr/queue, its own doc says "Mirrors app/client/src/pages/Downloads.jsx"). This is the true functional counterpart of web pages/Downloads.tsx, but it is still rendered in the OLD flat-list design:
- Page-level SectionHeader "Downloads" + subtitle (:30-34) — web has no page title, only section headers.
- _NeedsAttention (:78-130): renders each failing item as a flat MediaRow (alert leading icon, "Stuck" danger chip, subtitle = errorMessage or statusMessages joined, single Remove icon button). No danger-bordered card, no indexer·size mono line, no inline confirm, no "Remove & block"/blocklist.
- _ActiveDownloads (:132-221): flat MediaRow list. Each _DownloadRow (:184-221) = MediaThumb poster + title + subtitle + linear progress + mono "pct% · speed · seeds N" + pause/resume + delete. NO poster grid, NO circular ring-over-poster, NO aggregate "N active · ↓ · ↑" header, NO reconnecting indicator, NO up-speed. Delete calls actions.deleteTorrent(hash, deleteFiles:true) directly — NO confirm dialog and NO "also delete files" toggle.
- NO DownloadDetail overlay exists at all (no equivalent of DownloadDetail.tsx; no /downloads/:hash/detail fetch).
- Data wiring is present and correct: servarrDownloadsPollProvider polls downloads/enriched every 4s (servarr_provider.dart:364-392), servarrFailingQueueProvider polls radarr+sonarr every 6s (:411-442), ServarrQueueActions pause/resume/deleteTorrent/removeQueueItem (:446-466). ServarrDownload model (:334-359) has hash/name/subtitle/posterUrl/state/progress/size/downloaded/dlspeed/eta/numSeeds/isPaused/percent but LACKS upspeed, numLeechs(peers), kind. removeQueueItem cannot pass blocklist (ApiClient DELETE-body gap, noted at :461-463).

NATIVE side — flutter_app/lib/app/screens/downloads_screen.dart (route /downloads, the nav "Downloads" tab) + offline_screen.dart (/offline) + ui/widgets/download_button.dart, backed by state/downloads_provider.dart (CacheFillController) + state/offline_provider.dart + download/downloader.dart (legacy background_downloader, still referenced by providers.dart but downloads_provider now uses CacheFillController):
- downloads_screen.dart: page "Downloads" title + description, list of MediaRow rows with a status AppChip badge, progress bar, mono "pct% · received/total", pause/cancel, and a Retry row on failure. Roughly matches web DownloadProgress but uses AppChip badges instead of the inline mono state label + pulsing red active dot.
- offline_screen.dart: renders a POSTER GRID (GridView maxCrossAxisExtent 180, PosterCard tiles with Hero poster-<id> → /detail/:id, delete corner button, showConfirm dialog) and shows RUNTIME DURATION as the caption. This DIVERGES from web OfflineLibrary's flat list of "title + file size + trash".
- download_button.dart: matches web DownloadButton states well (Download / progress row with pause+cancel / Retry / "Downloaded" chip + remove). Minor visual deltas only.

Tokens already match the redesign's monochrome system (ui/tokens.dart:9-28: bg #0A0A0B, surface, dim/faint alpha ramp, near-white accent #F4F4F5, green #5AB98A sparingly, red #E0655E danger). So parity work here is composition/layout, not color/theming.
